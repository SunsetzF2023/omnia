# Omnia 版本历史

> 归档位置：`releases/Omnia-v{版本号}/`

---

## v2.1.0 (2026-07-10)

🔄 **回归单文件架构 — 移除游戏模块**

### 🔧 变更
- **CSS 全部内联回 index.html** — 不再依赖外部 CSS 文件，彻底解决加载问题
- **移除游戏模块** — 恢复为干净的 4 标签结构（cmd.book / 账本 / 日历 / 今日看点）
- **移除 Google Fonts CDN 依赖** — 离线可用，启动更快
- **禁用 HTTP 缓存** — 服务器加入 `Cache-Control: no-cache`

---

## v2.0.3 (2026-07-10)

🎨 **游戏 UI 统一 — Omnia 设计体系**

### ✨ 变更
- **游戏 CSS 完全重写** — 颜色、字体、间距、按钮、面板全部使用 Omnia 设计语言
- **移除 Google Fonts @import** — 不再依赖外部网络请求，使用 Omnia 的 JetBrains Mono 字体
- **修复 CSS 语法错误** — `#mod-game` 缺少 `{` 导致规则被浏览器忽略
- **修复全局泄露** — 所有游戏选择器均加上 `#mod-game` 作用域限制
- **统一视觉风格** — 游戏界面与笔记应用保持一致的深色仪表盘风格

---

## v2.0.2 (2026-07-10)

🐛 **紧急修复 — CSS 渲染/缓存问题**

### 🐛 修复
- **添加强制反缓存头** — HTTP 服务器添加 `Cache-Control: no-cache, no-store, must-revalidate`
- **添加内联关键 CSS** — 在 `<head>` 中加入后备样式，即使外部 CSS 加载失败也能保证布局正常
- **延迟加载 chart.js** — 使用 `defer` 属性，避免 CDN 脚本阻塞页面渲染

---

## v2.0.1 (2026-07-10)

🐛 **紧急修复 — 打包遗漏文件问题**

### 🐛 修复
- **修复** `electron-builder` 打包时遗漏 `styles.css`、`js/`、`game/` 目录的问题
- 之前安装后所有 CSS 样式丢失，界面退化为无样式 HTML
- 根本原因：`package.json` 的 `build.files` 只包含 3 个入口文件，排除了其他静态资源

### 📦 发布文件
- `Omnia Setup 2.0.1.exe` — Windows 安装包（修复版）

## v2.0.0 (2026-07-10)

🎉 **重大更新 — 全新架构 + 新名字**

### ✨ 新功能
- **就地编辑** — 查看笔记时直接点 ✎ 或双击步骤文本，原地修改
- **应用更名 Omnia** — 新名字、新起点

### 🔧 架构重构
- 代码从单个 `index.html`（1700行）拆分为模块化结构
- `js/` 目录下 6 个独立模块，改动互不影响

### 🐛 同步修复
- **队列机制**替代原有的 `isSaving` 锁，编辑不再丢失
- 同步失败时保留本地数据，不再清空界面
- 关闭窗口前自动保存到 Google Drive
- 账本保存增加错误提示

### 📦 发布文件
- `Omnia Setup 2.0.0.exe` — Windows 安装包
