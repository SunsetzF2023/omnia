# World 工作流指南

> 本文档规定了 `world/` 目录下所有内容的新增和修改的工作流程。
> 每个新 session 启动时，模型应首先读取本文件，确保遵循标准流程。

## 核心规则

**每次对 `world/` 目录下任何文件的新增或修改，完成后必须立即同步到 GitHub。**

永远确保 GitHub 上为最新版本的内容。

## 操作步骤

### 新增/修改文件后

```bash
# 1. 暂存所有 world/ 变更
git add world/

# 2. 提交（使用描述性的提交信息）
git commit -m "世界观更新：<简短描述>"

# 3. 推送到 GitHub
git push origin master
```

### 提交信息格式

```
世界观更新：<主要改动概述>

- <具体改动1>
- <具体改动2>
- ...
```

示例：
```
世界观更新：新建红尘君等八君角色，更新小柳洲战局

- 新建：红尘君、神梦君、鬼嘲君（八君之三）
- 更新：小柳洲（幻阵对峙）、柳家（封锁战）、血刀门（八君动态）
```

## 重要提醒

- **无需 bump 版本号** — 世界观内容更新不涉及应用代码，不需要发布新版本
- **无需打 tag** — 不需要创建 GitHub Release
- **直接 push master** — 世界观内容可以直推主分支
- **每次修改都要提交** — 不要积攒多轮对话的内容一起提交。每轮世界构建结束后立即 push

## 读取顺序

新 session 中，模型应：

1. 先读 `world/_INDEX.md` 了解当前世界观全貌
2. 再读 `world/README.md` 了解目录结构
3. 最后读本文件 `world/WORKFLOW.md` 了解工作流程
